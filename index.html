Basics of the web are: the web, html, url, http and web application.

HTML

HTML is hypertext markup language, which is the MAIN type of content used. This content tells you “what you see”. Markup tells you "how it looks”. These documents are the main type of content used one the web.

The major pieces of the web are: You, computer and browser, the internet, servers, and HTTP (which are protocols).

Lesson 2: Creating a Structured Document with HTML

HTML

HTML is considered the foundation of a page, while CSS is the decoration and style of a page. JS is the interactive components.

The "Tree-Like structure" of HTML

Tree-like Structures (branching structures) of HTML. A tree-like structure is like the branches on a tree. Different branches with different styles. The structures appear as boxes. Boxes inside of boxes inside of more boxes.These boxes consist of an HTML,CSS, and DOM.

Text Editors

Programmers have the ability to alter the branches via text editors such as, Sublime Text, CodePen, and scratchpad.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

It is very important that programmers avoid repetition for many reasons.

Avoiding Errors:When a programmer does not avoid repetition, they do the same thing over and over. Then when a programmer decides to make changes it tends to lead to errors. It is important that programmers make the same change to all the pertinent areas to avoid errors.

Consistency

Consistency is key to programming codes. When a programmer specifies the same style for similar codes. This makes it easier to make changes. For example applying the class code of "class".

Understanding CSS

CSS stands for Cascading Style Sheets and CSS allows programmers to have control of how their HTML docs will look. CSS can be thought of as search and replace tools. Cascading means there are rules that are applied to an element that is a direct match, as well as they are applied to the child elements, unless the child element has multiple overlapping rules that are defined to it then those rules take affect, this is call "Inheritance".

Box Sizing and Positioning

As we learned in a previous lesson, HTML elements are boxes, each box has four components. Being as there are so many components to each box it can be difficult to size the box just right.
There are two techniques that are useful with sizing, one is to set the size in percentage as opposed to pixels, the other is to utilize the box sizing border attribute.
Positioning: Div's are block elements, and by default they use the entire page. Adding rules of display flex in the CSS will override the default and allow the DIV's to appear next to each other.
